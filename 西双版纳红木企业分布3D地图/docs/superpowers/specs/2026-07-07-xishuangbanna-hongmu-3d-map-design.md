# 西双版纳红木企业分布 3D 地图设计

## 背景与目标

本项目要制作一个“可交互 3D 地图 + 数据看板”的本地网页应用，用于展示西双版纳红木企业、家具城、门店、加工厂及相关机构的空间分布。首版数据以用户提供的 `西双版纳红木企业不完全整理，部分附带谷歌地图电话号码.docx` 为主，结合已生成的平面分布图和谷歌地图截图作为视觉与空间参考，并为后续公开检索补充预留数据字段。

核心目标是“细致完整”：所有已整理企业都必须保留在数据集中和界面中，不因为地址不精确而丢弃。地图需要明确区分精确位置、道路级位置、片区/县市级位置和待核查位置，避免把概略地址误表现为精确门牌。

## 已确认范围

- 成品形态：A+B，即可交互 3D 地图与数据看板结合。
- 底图方式：离线 3D 看板为主，企业详情中提供外部地图搜索/核查链接。
- 数据策略：完整保留所有企业记录，同时标注定位精度和来源。
- 首版数据概况：Word 表中共 78 条记录，其中约 9 条精确/近似精确、21 条道路级、47 条片区/县市级、1 条待核查，49 条带电话。
- 重点区域：景洪主城区为主展示对象，并保留勐腊、勐海及其他区域的分布层。

## 用户体验

应用首屏直接进入地图工作台，不做营销落地页。主体为左侧/中部 3D 地图，右侧为数据看板与企业列表。用户可以旋转、缩放、平移地图，点击 3D 点位或聚合柱查看企业详情。

地图表现分为三层：

- 精确/近似精确：独立点位或较细的柱体，适合点击查看单个企业。
- 道路级：沿对应道路或道路片区形成聚合点/柱，详情中列出该道路下的企业。
- 片区/县市级：落在景洪、勐腊、勐海等区域中心或片区聚合位置，作为完整性标记展示。
- 待核查：使用独立视觉样式提醒该记录需要后续人工确认。

看板展示企业总数、各区域数量、类型分布、定位精度分布、带电话记录数量，以及当前筛选后的动态统计。企业详情展示名称、拼音、类型、地址、电话、来源、定位精度、备注和外部地图搜索链接。

## 数据模型

每条企业记录至少包含以下字段：

- `id`：序号或内部唯一标识。
- `name`：中文名称。
- `pinyin`：拼音/英文名。
- `type`：企业或门店类型。
- `address`：原始地址文本。
- `phone`：电话，缺失时为空。
- `region`：景洪市、勐腊县、勐海县、其他或待判定。
- `locationPrecision`：`exact`、`road`、`area`、`unknown`。
- `source`：Word、平面图、公开检索、人工补充等。
- `coordinates`：可选，经纬度或本地示意坐标。
- `externalSearchUrl`：外部地图搜索链接。
- `notes`：核查说明或重复合并说明。

首版会从 Word 表自动抽取 78 条记录，保留原始文本。若后续检索到新增企业，将作为新增记录进入同一数据表，来源标为公开检索，并与同名/近似同名记录做人工可审查的去重，不直接覆盖原始资料。

## 地图与可视化设计

离线 3D 地图采用示意化地理底板：西双版纳区域轮廓、澜沧江/主要水系、景洪主城区、勐腊、勐海等区域标注，以及民航路、勐泐大道、景德路、环城南路等数据中高频道路。该底板不声称是测绘级底图，而是用于清晰表达企业分布、聚集带与数据完整性。

3D 点位编码：

- 颜色区分类型：家具城/卖场/商店、工厂/加工厂、协会/公司/机构、礼品/根雕/工艺品等。
- 高度表达聚合数量：道路级和片区级聚合柱高度随企业数量变化。
- 形状或描边表达定位精度：精确点、道路级聚合、片区级聚合和待核查记录具有明显差异。
- 选中状态突出显示，并联动右侧企业列表。

景洪主城区提供放大视图，重点呈现民航路、勐泐大道、曼弄枫/会展片区等聚集带。用户可以在全州视图和主城区视图之间切换。

## 交互与筛选

首版需要支持：

- 区域筛选：全部、景洪、勐腊、勐海、其他。
- 类型筛选：家具城/卖场/商店、工厂/加工厂、公司/机构、礼品/根雕/工艺品、其他。
- 定位精度筛选：精确、道路级、片区级、待核查。
- 电话筛选：全部、带电话、缺电话。
- 文本搜索：按企业名称、地址、电话检索。
- 点击地图对象：打开详情面板或聚合列表。
- 外部核查：详情中一键打开地图搜索链接。

筛选时地图点位、聚合柱、统计卡片和企业列表同步更新。聚合对象点击后应列出其包含的企业，不能只显示一个代表名称。

## 架构

首版实现为本地静态网页应用：

- `index.html`：应用入口。
- `src/`：界面、交互、3D 场景和数据处理逻辑。
- `data/enterprises.json`：结构化企业数据。
- `assets/`：用户提供的平面图、地图截图及后续视觉资产。
- `docs/`：设计文档和数据说明。

技术上优先使用浏览器原生能力与 Three.js。若项目依赖可本地安装，则用 Vite 管理开发服务器；若网络或依赖安装受限，则采用可直接打开的静态 HTML 方案，并通过本地文件内嵌或 vendored 依赖保证可运行。

3D 场景、数据筛选和详情列表应拆分为独立模块，避免把所有逻辑堆在单个文件中。数据解析脚本可以单独保留，用于从 Word 重新生成 JSON，保证后续资料更新时能复用。

## 数据流

1. 从 Word 表抽取原始行，生成结构化初始数据。
2. 根据地址关键词推断区域、定位精度和示意坐标。
3. 对道路级和片区级记录生成聚合组，同时保留每条企业明细。
4. 前端加载 JSON 后构建全量索引。
5. 用户操作筛选器或搜索框后，生成当前视图数据。
6. 地图、看板、列表和详情面板消费同一份当前视图数据。

## 错误处理与数据可信度

缺少电话、地址过粗、疑似重复、无法判断区域的记录不会被删除，而是在界面中显示为缺失或待核查。外部地图搜索链接基于企业名称和地址生成，作为核查入口，不代表位置已确认。

对于公开检索补充的数据，需要记录来源和检索日期。若新增记录与 Word 中已有企业同名或高度相似，先保留为疑似重复状态，待人工确认后再合并。

## 测试与验收

数据验收：

- Word 中 78 条记录全部进入 `enterprises.json`。
- 每条记录都有 `id`、`name`、`type`、`address`、`region`、`locationPrecision` 和 `source`。
- 统计结果与数据文件一致：总数、电话数量、区域数量、定位精度数量可复算。

界面验收：

- 本地打开或启动开发服务器后可以看到 3D 地图与数据看板。
- 筛选、搜索、点击点位、点击聚合对象、查看详情均可用。
- 精确点、道路级聚合、片区级聚合和待核查记录视觉上可区分。
- 景洪主城区能作为重点视图查看。
- 企业详情包含外部地图搜索链接。

视觉验收：

- 整体风格参考用户提供的红木主题平面图，但界面保持清晰、可读、适合反复筛选。
- 不使用纯装饰性背景干扰地图和列表。
- 桌面端和移动端文本不重叠，按钮和详情面板内容不溢出。

## 非目标

- 首版不承诺所有企业都已获得精确经纬度。
- 首版不依赖付费地图 API 或商业地图 key。
- 首版不做测绘级行政边界或道路精度声明。
- 首版不直接删除疑似过期企业，只标注待核查。

## 后续扩展

后续可以加入坐标核查表、公开企业名录补充、批量地理编码、导出 PNG/PDF、演示模式、时间维度更新记录，以及更精细的真实地图叠加层。
